# 관내여비 콕검 v0.2

에듀파인에서 출력한 관내여비 내역(`.xls`, `.xlsx`)을 브라우저에서 읽고, 출장지 위치 확인과 보행 왕복거리 계산을 한 번에 진행해 왕복 2km 이내 출장과 확인이 필요한 출장만 골라주는 급여담당자용 업무 보조 웹앱입니다.

> 이 앱은 검토 대상을 좁혀 주는 보조 도구입니다. 최종 지급 여부는 관련 지침과 기관 기준을 확인해야 합니다.

## v0.2 핵심 업데이트

- 메인 CTA를 **일괄검사 시작**으로 변경
- 한 번의 실행으로 다음 과정을 연속 처리
  1. 저장된 출장지 위치 적용
  2. 명확한 출장지 자동 검색·확정
  3. 모호한 출장지는 확인 필요로 분리
  4. 위치가 확인된 출장지의 왕복 보행거리 계산
  5. 왕복 2km 이내·초과·경계 구간 자동 분류
- 일괄검사 진행률과 현재 처리 중인 출장지 표시
- 검사 중지 기능: 이미 완료된 결과 유지
- 미완료 항목만 다시 검사
- 전체 거리 다시 계산
- 검색 실패와 거리 계산 실패를 별도 상태로 표시
- 출장지별 결과와 출장 건별 결과 분리
- 기존 여비 지급 내역과 함께 검토
- 왕복 2km 이내인데 지급금액 또는 여비금액이 있는 경우 **지급내역 확인 필요** 표시
- 결과 필터 추가
  - 확인 필요
  - 왕복 2km 이내
  - 왕복 2km 초과
  - 경계 구간
  - 계산 실패
  - 기존에 여비가 지급된 건만 보기
  - 부지급 표시가 없는 건만 보기
- 결과 엑셀 파일명과 시트 구성 개선

## 기존 기능

- 에듀파인 `.xls`·`.xlsx` 업로드
- 인쇄용 페이지 표시, 기관명, 합계·소계 행 자동 제외
- 같은 출장지 자동 묶기
- 주소가 포함된 출장지는 주소 우선 검색
- `인근`, `주변`, `일대`, `협의회 장소` 등 모호한 출장지는 자동 확정하지 않음
- 근무지와 사용자가 확정한 출장지 위치를 브라우저에 저장
- TMAP 보행자 경로 API로 갈 때·올 때를 각각 계산
- 왕복 2,000m 기준 및 1,900~2,100m 경계 구간 표시
- 결과 복사 및 `.xlsx` 내려받기

## 개인정보 처리

- 업로드한 엑셀 파일은 브라우저에서만 읽으며 서버에 저장하지 않습니다.
- 지도 API에는 장소 검색어 또는 출발·도착 좌표만 전달합니다.
- 출장자명, 출장목적, 여비금액, 지급금액, 출장일은 지도 API로 보내지 않습니다.
- 근무지와 사용자가 확정한 출장지 위치는 해당 브라우저의 `localStorage`에 저장됩니다.

## Vercel 업데이트 방법

기존 v0.1.1 저장소를 사용하는 경우 새 저장소를 만들 필요가 없습니다.

1. 이 ZIP의 압축을 풉니다.
2. 기존 GitHub 저장소의 파일을 v0.2 파일로 교체합니다.
3. GitHub에 커밋·업로드합니다.
4. Vercel이 자동 재배포되거나, Vercel의 Deployments에서 Redeploy를 실행합니다.
5. 기존 `TMAP_APP_KEY` 환경변수는 그대로 사용합니다.

저장소 루트에는 다음 파일과 폴더가 바로 보여야 합니다.

```text
api/
src/
tests/
index.html
package.json
vercel.json
vite.config.js
```

## TMAP API 설정

1. SK Open API에서 앱을 생성합니다.
2. PRODUCTS에서 TMAP 무료 요금제를 선택하고 `관내여비 콕검` 앱에 연결합니다.
3. 앱의 App Key를 복사합니다.
4. Vercel 프로젝트의 **Settings → Environment Variables**에 등록합니다.

```text
Name: TMAP_APP_KEY
Value: 발급받은 App Key 값만 입력
Environment: Production and Preview
```

5. 환경변수를 추가하거나 변경한 뒤 반드시 새로 배포합니다.

API 키는 `api/` 서버리스 함수에서만 사용하므로 프론트엔드 번들에 직접 포함되지 않습니다.

## 로컬 개발

### 패키지 설치

```bash
npm install
```

### 화면 개발 서버

```bash
npm run dev
```

지도 API까지 로컬에서 확인하려면 Vercel CLI 개발 서버와 `.env.local`의 `TMAP_APP_KEY`가 필요합니다.

```bash
npx vercel dev
```

### 프로덕션 빌드

```bash
npm run build
```

빌드 결과는 `dist` 폴더에 생성됩니다.

## 검증

```bash
npm test
npm run build
```

포함된 비식별 테스트 파일 기준:

- `.xls`와 `.xlsx` 동일 결과
- 관내출장 4건
- 출장자 4명
- 출장지 3곳
- 결과 엑셀 3개 시트 생성 확인
- API 키 미설정 오류 처리 확인

실제 업로드해 주신 에듀파인 파일도 별도로 검증했습니다.

- `.xls`: 관내출장 30건 · 출장자 16명 · 출장지 17곳
- `.xlsx`: 관내출장 30건 · 출장자 16명 · 출장지 17곳
- 두 파일의 출장지별 건수도 동일

실제 파일은 개인정보 보호를 위해 프로젝트 ZIP에 포함하지 않습니다.

## 결과 엑셀

파일명:

```text
관내여비콕검_일괄검사결과_YYYYMMDD_HHMM.xlsx
```

시트:

1. `출장별 검사결과`
2. `출장지별 거리`
3. `확인 필요`

## 주요 파일

```text
api/
  _tmap.js            TMAP 공통 호출·키 보호
  tmap-health.js      API 키와 상품 연결 상태 확인
  tmap-search.js      장소·주소 검색
  tmap-route.js       갈 때·올 때 보행경로 계산
src/
  parser.js           에듀파인 출력 양식 파싱
  api.js              프론트엔드 API 호출
  storage.js          근무지·출장지·거리 캐시 저장
  exporter.js         일괄검사 결과 엑셀 생성
  main.js             일괄검사·중지·재검사·화면 상태
  styles.css          반응형 UI
```
