# 관내여비 콕검 v0.3

에듀파인에서 출력한 관내여비 `.xls`·`.xlsx` 파일을 브라우저에서 분석하고, 출장지의 보행 왕복거리와 출장시간을 확인해 급여담당자의 지급 검토 대상을 정리하는 웹앱입니다.

## v0.3 핵심

- 동일 출장지를 묶어 TMAP 보행 왕복거리 일괄검사
- 출장자와 날짜가 같은 내역을 한 묶음으로 정리
- 출장시간 자동 계산
- 왕복 2km 이내, 경계 구간, 위치·거리 미확인, 당일 여러 건 우선 표시
- 출장별 실제 이동수단 입력
- 검토 완료·보류·다시 검토 상태 저장
- 사람·날짜별 검토 메모 저장
- 사람·날짜별 지급검토, 출장별 검토결과, 출장지별 거리, 확인 필요 시트로 엑셀 출력

앱은 지급액이나 지급 오류를 확정하지 않습니다. 거리·시간·이동수단과 기존 지급내역을 한 화면에 모아 최종 확인 대상을 좁히는 보조 도구입니다.

## 지원 파일

- `.xls`
- `.xlsx`

에듀파인 인쇄용 출력물에 포함된 제목, 페이지 번호, 학교명, 반복 머리글, 합계·소계 행을 제외하고 실제 관내출장 행만 읽습니다.

## 로컬 실행

```bash
npm install
npm run dev
```

지도 API까지 로컬에서 사용하려면 Vercel 개발환경 또는 동일한 `/api` 서버리스 함수 실행환경이 필요합니다.

## Vercel 배포

1. 이 폴더의 파일을 GitHub 저장소 루트에 올립니다.
2. Vercel에서 저장소를 새 프로젝트로 가져옵니다.
3. 환경변수에 아래 값을 등록합니다.

```text
TMAP_APP_KEY=발급받은_App_Key
```

환경은 `Production and Preview`를 선택하면 됩니다.

4. 배포하거나 환경변수 추가 후 다시 배포합니다.

## 개인정보 처리

- 업로드 파일은 브라우저에서만 읽고 서버에 저장하지 않습니다.
- 지도 API에는 장소 검색어와 출발·도착 좌표만 전달합니다.
- 출장자명, 출장목적, 출장일, 지급금액은 지도 API로 보내지 않습니다.
- 근무지, 확인한 출장지, 거리 캐시, 이동수단, 검토 상태와 메모는 사용자 브라우저의 `localStorage`에 저장됩니다.

## 테스트

```bash
npm test
npm run build
```

테스트 범위:

- `.xls`·`.xlsx` 파싱 결과 일치
- 사람·날짜별 묶음과 출장시간 계산
- 지급 검토 결과 엑셀 생성
- 지도 API 오류 응답 처리

## 주요 폴더

```text
tests/                비식별 테스트와 검증 코드
```
