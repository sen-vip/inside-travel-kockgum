# 관내여비 콕검 v0.1.1

에듀파인에서 출력한 관내여비 내역(`.xls`, `.xlsx`)을 브라우저에서 읽고, 같은 출장지를 묶어 근무지와 출장지 사이의 보행 왕복거리를 확인하는 업무 보조 웹앱입니다.


## v0.1.1 수정 사항

- TMAP 인증·상품 사용 오류를 빈 검색 결과로 잘못 표시하던 문제 수정
- 환경변수에 공백, 따옴표 또는 `TMAP_APP_KEY=` 접두사가 포함되어도 앱 키를 정리해 사용
- 장소 검색에 `searchType=all`을 명시하고 검색 결과 수 확대
- 앱 시작 시 키 존재 여부가 아니라 실제 TMAP 연결·상품 사용 가능 여부 확인
- 인증 실패 시 Vercel 환경변수와 SK open API 상품 사용 신청을 확인하도록 정확한 오류 표시

## v0.1 핵심 기능

- 에듀파인 관내여비 내역 `.xls`·`.xlsx` 업로드
- 인쇄용 페이지 표시, 기관명, 합계 행 자동 제외
- 같은 출장지 자동 묶기
- 주소가 포함된 출장지의 주소 우선 검색
- `인근`, `주변`, `일대`, `협의회 장소` 등 모호한 출장지 자동 확정 금지
- 근무지와 출장지 위치를 브라우저에 저장
- TMAP 보행자 경로 API로 갈 때·올 때를 각각 계산
- 왕복 2,000m 기준 및 1,900~2,100m 경계 구간 표시
- 출장별 결과표·필터·결과 복사
- 결과 `.xlsx` 내려받기(출장별 결과 / 출장지별 거리 / 확인 필요)

> 이 앱은 급여담당자가 확인할 출장 건을 좁혀 주는 보조 도구입니다. 최종 지급 여부는 관련 지침과 기관 기준을 확인해야 합니다.

## 개인정보 처리

- 업로드한 엑셀 파일은 브라우저에서만 읽으며 서버에 업로드하지 않습니다.
- 지도 API에는 장소 검색어 또는 출발·도착 좌표만 전달합니다.
- 출장자명, 출장목적, 여비금액, 지급금액은 지도 API로 보내지 않습니다.
- 근무지와 사용자가 확정한 출장지 위치는 해당 브라우저의 `localStorage`에 저장됩니다.

## 실행 방법

### 1. 패키지 설치

```bash
npm install
```

### 2. 화면만 확인

```bash
npm run dev
```

Vite 개발 서버에서는 정적 화면과 엑셀 분석 기능을 확인할 수 있습니다. 지도 API까지 로컬에서 함께 확인하려면 Vercel CLI의 개발 서버를 사용합니다.

```bash
npx vercel dev
```

### 3. 프로덕션 빌드

```bash
npm run build
```

빌드 결과는 `dist` 폴더에 생성됩니다.

## TMAP API 설정

1. SK Open API에서 앱을 생성하고 TMAP 상품을 사용 설정합니다.
2. 앱 키를 발급받습니다.
3. Vercel 프로젝트의 **Settings → Environment Variables**에 아래 변수를 등록합니다.

```text
TMAP_APP_KEY=발급받은_앱키
```

4. 다시 배포합니다.

API 키는 `api/` 서버리스 함수에서만 사용하므로 프론트엔드 번들에 직접 포함되지 않습니다.

## Vercel 배포

1. 이 프로젝트를 GitHub 저장소에 올립니다.
2. Vercel에서 저장소를 가져옵니다.
3. Framework Preset은 Vite로 인식됩니다.
4. `TMAP_APP_KEY` 환경변수를 등록합니다.
5. Deploy를 실행합니다.

`vercel.json`에 빌드 출력 폴더와 서버리스 함수 제한 시간이 설정되어 있습니다.

## 샘플 파일 검증

개발 시 다음 명령으로 업로드한 샘플 `.xls`와 `.xlsx`가 동일하게 파싱되는지 확인할 수 있습니다.

```bash
npm run test:parser
```

프로젝트에 포함된 비식별 테스트 파일은 다음 결과로 검증됩니다.

- 관내출장 4건
- 출장자 4명
- 출장지 3곳
- `.xls`와 `.xlsx`의 출장 데이터 일치

실제 업로드해 주신 에듀파인 파일도 별도로 검증했으며, 두 형식 모두 **관내출장 30건 · 출장자 16명 · 출장지 17곳**으로 동일하게 인식되었습니다. 실제 파일은 개인정보 보호를 위해 프로젝트 ZIP에 포함하지 않습니다.

다른 경로의 파일을 시험하려면 파일 경로를 인수로 넘깁니다.

```bash
node tests/parser.test.mjs "C:/path/sample.xls" "C:/path/sample.xlsx"
```

## 주요 파일

```text
api/
  _tmap.js            TMAP 공통 호출·키 보호
  tmap-health.js      API 키 설정 상태 확인
  tmap-search.js      장소·주소 검색
  tmap-route.js       보행 왕복거리 계산
src/
  parser.js           에듀파인 출력 양식 파싱
  api.js              프론트엔드 API 호출
  storage.js          근무지·출장지·거리 캐시 저장
  exporter.js         결과 엑셀 생성
  main.js             화면·상태·작업 흐름
  styles.css          반응형 UI
```
